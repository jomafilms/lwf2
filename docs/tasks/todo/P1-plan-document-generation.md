# P1-4: Plan Document Generation

> **Status:** TODO
> **Priority:** P1
> **Depends on:** P1-1 (property saving), P0-4 (plant cards)
> **Blocks:** P2-3 (compliance documents)

## Problem

Users build plans in the app but can't export them. Need a downloadable/printable document showing: property with zone map, recommended plants per zone, estimated costs, nursery sources. This is what a homeowner hands to a landscaper or submits to an HOA.

## Proposed Changes

### 1. Plan document API
- `GET /api/plans/[id]/document` — generates HTML document (server-rendered)
- `GET /api/plans/[id]/document?format=pdf` — PDF via puppeteer or html-pdf (stretch)

### 3. "Download Plan" button on map page
- Appears after user has zones + plants selected
- Generates and downloads the document
